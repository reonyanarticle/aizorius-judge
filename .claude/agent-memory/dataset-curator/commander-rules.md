commander候補25問（`evaluation/candidates/commander.json`）をCR 2026-06-19 (en) で検証した際の根拠対応。番号は [[cr-version]] にも実在記録。

## 裁定根拠マップ
- 統率者の指定条件: 903.3（指定はカード自体の属性、ゾーン移動で保持）／903.3a（「統率者になれる」能力）。**プレインズウォーカーは通常Commanderの統率者不可**（Brawlは903.12cで別途可）。
- 統率者税: 903.8。追加コスト＝過去に**統率領域から**唱えた回数×{2}（1回目0, 2回目+2, 3回目+4）。手札等他領域からの唱えは税に無関係・回数にも数えない。
- 統率者税は「additional cost」と903.8が明言 → 代替コスト（マナ・コスト不払い）でも消えない。総コスト計算は601.2f、代替コストは118.9。
- 統率者ダメージ21点: 903.10a と 704.6c（**同一SBAが両所に記載**）。「combat damage」限定=非戦闘ダメージは数えない。
- 色アイデンティティ: 903.4（定義）／903.4a（ゲーム開始前に確定、ゲーム中の色変更は不影響）／903.4c（リマインダー文無視）／903.4d（両面カードは裏面も含む=712.8aの例外）。CDA=604.3, 色指標=204.1。
- ファイレクシア・マナは有色: 107.4f（{W/P}は白）→色アイデンティティに算入。ライフ払い可否は無関係。
- デッキ構築: 903.5a（ちょうど100枚）／903.5b（基本土地以外は英語名相違）／903.5c（色アイデンティティ包含）／903.5d（基本土地タイプは出す各色が統率者の色アイデンティティ内）／903.5e（サイドボードなし）。
- 統率領域への移動: 903.9a（墓地・追放→統率領域はSBA、オーナー任意。704.6dに重複記載）／903.9b（手札・ライブラリー行きは置換、複数回適用可=614.5の例外）。戻す判断はオーナー(108.3)。
- 開始: 903.6（統率領域へ配置＋ライブラリー化）／903.7（ライフ40・手札7）。
- 多人数デフォルト: 903.2（Free-for-All＋複数攻撃オプション、影響範囲限定なし）→806.1。
- meld統率者: 903.3b（合体後パーマネントが統率者）。merged=903.3c。
- 共闘partner: 702.124a/b（2枚指定・計100枚・両方統率領域）／702.124c（色アイデンティティは2人の和）／702.124d（税は独立してカウント＋21点ダメージも別々にカウント）。

## 落とし穴・注意
- 903系サブルールは版で動きやすい。2026-06-19では 903.10a=21点SBA, 903.9a/9b=領域移動。**必ずCRで再確認**。
- 21点SBAは903.10aと704.6cの二重記載。rules_citedはどちらか一方でも、retrieval_relevant_rulesには両方入れると網羅的。
- 「104」等セクションのみの番号はvalidatorで実在扱いされない（個別番号のみ）。104はサブ番号(104.2a等)を使う。

## 要人間確認として notes に残した問（候補には採用済み、内容確認推奨）
- commander-016（統率者のコピー・トークンは統率者でない/統率者ダメージに数えない）: CRに直接条文なし。903.3「指定はカードの属性」からの標準的裁定として採用。
- commander-024（統率者税は代替コストでも支払う）: 903.8の「additional cost」文言＋601.2f/118.9からの導出。明示の統合条文はない。

## 未採用（自信不足で候補外にした題材）
- Companion（702.139/103.2b）とCommander戦の相互作用: 起動タイミング・starting deck定義が絡み確信不足。
- 統率者がコピー効果で別クリーチャーになった場合の「same commander」判定: 明示条文が薄く保留。
