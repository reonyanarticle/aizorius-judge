CR 2026-06-19 (en) で本文照合済みの戦闘関連番号。**版ズレ注意点あり**（旧知識と食い違う）:

- **警戒 = 702.20**（702.20b「攻撃してもタップしない」）。**702.21 は Ward（護法）** であり Vigilance ではない。旧版の番号感覚で 702.21 を警戒に使わないこと。
- **威迫（menace）= 702.111**（702.111b「2体以上でしかブロックできない」）。
- 飛行 702.9（702.9b）／到達 702.17（702.17b）— 相互参照 See rule で結ばれている。両方を retrieval に入れる。
- 先制攻撃 702.7（702.7b）／二段攻撃 702.4（702.4b）。両方とも本文が 510.4 とほぼ同文で、2回の戦闘ダメージ・ステップを規定。
- トランプル 702.19：702.19b（致死ダメージ後に超過を割り振る）、702.19d（ブロック後ブロッカー消滅時は全ブロッカーに致死割り振り済み扱い）、702.19c は「トランプル・オーバー・プレインズウォーカー」。
- 接死 702.2：702.2b（SBAで破壊）、702.2c（接死の1点以上は超過判定上「致死ダメージ」扱い→トランプルと結合すると1点で致死）。

**版ズレ/概念消失の重要点**:
- 現行CRに **「ダメージ割り振り順（damage assignment order）」の概念は存在しない**（grepで0件）。複数ブロック時は 510.1c で「コントローラーが望むように分ける」だけ。致死ダメージ順の制約はトランプル(702.19b)専用。旧ルール知識で「順に致死を割り振る」と書かない。

その他:
- ブロック指定 509.1a（各ブロッカーは攻撃1体を選ぶ＝1体は1体しかブロックできない／自分・自分のPW・自分が守る戦道を攻撃するものだけブロック可）。
- 509.1h（一度ブロックされた攻撃クリーチャーは全ブロッカー消滅後もブロック済みのまま→510.1cで無ブロッカーなら割り振り無し。トランプル無しはダメージ通らない）。
- 506.4c（攻撃対象PW/戦道が取り除かれても攻撃クリーチャーは戦闘に残るが何も攻撃していない→510.1bで割り振り無し）。
- 戦闘ダメージ同時 510.2、致死SBA 704.5g（マークダメージ≥タフネス）、接死SBA 704.5h。
