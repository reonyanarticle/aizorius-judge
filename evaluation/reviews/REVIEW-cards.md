## 1. カテゴリ設計の提案

### カテゴリ名: `card_interactions`
- 個別カード名を用い、そのカードとルールの**相互作用**を問う。既存110問（すべてルール一般）に無い領域。
- **問数**: 初回15問（要望どおり）。承認後の拡充目安は 15〜20問。
- **対象カード選定方針**: 有名で裁定が確立し、かつ**誤解が多い相互作用**を持つカードに限定。
  カードテキストを読めば分かる問い（例: 単なるP/Tやマナ・コスト）は避け、CR外（イベント規定等）も除外。
  統率者戦の定番相互作用（ドラニスの判事×統率領域、敵対工作員）を含める。

### スキーマ拡張の提案（要人間判断）
既存スキーマ（`docs/EVALUATION.md` §3）に **`cards` フィールドを追加（任意）** することを提案する。
- 形式: `"cards": [{"ja": "血染めの月", "en": "Blood Moon"}, ...]`（日本語名＋英語名の配列）。
- 追加理由:
  1. `lookup_card` / `get_card_rulings` に渡す**カード名を機械可読に明示**でき、ツール評価の入力を固定できる。
  2. 検証に使った Scryfall カードを記録でき、再検証・版追随が容易。
  3. **任意フィールドなので既存110問に影響しない**（後方互換の追加のみ）。
- 代替案: `cards` を足さず question 本文のカード名だけで運用することも可能だが、日本語表記ゆれ
  （例「ブロコス」対「ブロッコス」）で lookup_card 入力が不安定になるため、明示フィールドを推奨する。
- **この拡張は EVALUATION.md §3 のスキーマ表への追記を伴う**ため、承認後に別途反映する（本候補では
  ドキュメントは未編集）。

### 検索単体評価（recall@5）との関係 — 除外を提案
- これらの問題は **`lookup_card` が主役**で、`search_rules` は補助（結論の CR 根拠づけ）である。
- そのまま `tests/test_search.py` の recall@5 母集団に混ぜると、**検索単体指標が歪む**
  （カード検索前提の問を検索エンジンだけで測ることになる）。
- **提案**: 検索単体評価は `category != "card_interactions"` でフィルタして**除外**する。
  ただし各問に `retrieval_relevant_rules` は付けてあるので、「カード相互作用問の中でルール検索が
  どれだけ根拠を拾えるか」を**別建ての補助指標**として測ることは可能（本流の recall@5 とは分離）。
- 統合評価（第3層）には含めてよい（`lookup_card`＋`search_rules`＋推論の最終品質を測る対象）。

## 3. 出典検証の要点（版ずれ・落とし穴）

- **キーワードの版依存番号を再照合済み**: 変容=702.140 / 相棒=702.139 / 大群=701.47 / 灯争=保留なし。
  護法(702.21)・威迫(702.111)との混同は起きていない。
- **レイヤー**: 種類変更は第4層 = **613.1d**（血染めの月）。CDA の P/T は第7層(613.4)で適用、
  P/T以外は 613.3 で最初に適用（タルモゴイフ）。
- **ドロー定義は 121（120ではない）**。CR 2026-06-19 では「Drawing a Card」は 121.1/121.2。
- **統率者戦の相互作用**（002 ドラニスの判事）: 統率領域からの唱えは 903.8 の**許可**であり、
  「手札以外から唱える」ことを禁止する効果に上書きされる（601.3a）。統率者税を払っても不可。
- **忠誠度カウンター**（013）: 「コストとして置く」ものは効果ではないため置換(614.1)の対象外。
  この区別が誤答ポイント。根拠は 606.5（忠誠度能力のコスト）＋306.5b（初期忠誠度＝効果）。
- **破壊不能の限界**（015）: 破壊不能は破壊・致死・接死は無効(702.12b)だが、**生け贄(701.21)**と
  **タフネス0以下(704.5f)**は防げない。「破壊不能なら何でも生き残る」は禁止誤答に指定済み。

## 4. 要人間確認（自信度の注記）

以下は CR に直接の統合条文がなく、Scryfall 公式裁定＋複数条文からの標準的な導出で組んだ。
内容は確立した裁定だが、番号の**帰属**を人間が最終確認することを推奨する。

- **010 エムラクール**: 「唱えたとき」の誘発イベントを 601.2i（呪文が唱えられた瞬間）に帰属させ、
  スタック配置を 603.3 とした。CR に「cast-trigger は本体より先に解決」という一文はなく、
  601.2i＋603.3＋スタックの後入れ先出しからの導出。裁定自体は Scryfall で明示確認済み。
- **004 オークの弓使い**: 「各ドロー・ステップの最初の1枚を除く」の除外自体はカード固有テキスト。
  ルール根拠は「ドローの定義(121.1)」と「複数枚は1枚ずつ(121.2)」に置いた。amass(701.47)は
  周辺 retrieval に含めたが結論の核ではない。
- **013 倍増の季節**: 606.5 は「忠誠度能力の総コスト」に関する条文で、「コストで置く忠誠度は
  倍増しない」を直接述べてはいない。コスト≠効果ゆえ 614.1 が及ばない、という導出。
